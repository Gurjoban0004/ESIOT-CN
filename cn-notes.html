Introduction & Topologies
Uses of Networks: Resource sharing, communication, cost reduction, reliability via redundancy, scalability.
  • Bus: Single cable backbone. Simple, cheap. Collision-prone. Break = full outage.
  • Star: All nodes → central hub/switch. Easy to manage. Hub fail = full outage.
  • Ring: Each node → next, forming loop. Token passing. One break disrupts ring.
  • Mesh: Every node connected to every other. Full mesh = n(n−1)/2 links. Very reliable.
  • Hybrid: Combination of two or more topologies.
Collision Domain: Segment where collisions can occur. Switches create 1 collision domain per port. Hubs share 1 domain.
Broadcast Domain: Area where broadcasts are forwarded. Routers separate broadcast domains. VLANs also separate them.
  • Hub — L1: 1 collision domain, 1 broadcast domain
  • Switch — L2: N collision domains (1/port), 1 broadcast domain
  • Router — L3: Separates BOTH collision AND broadcast domains

OSI vs TCP/IP Differences

  • OSI = 7 layers (theoretical model); TCP/IP = 4 layers (practical)
  • OSI: Session & Presentation separate; TCP/IP: merged into Application
  • OSI: Data Link & Physical separate; TCP/IP: merged as Network Access
  • TCP/IP developed alongside internet; OSI is general reference model
  • PDU names: Bits → Frames → Packets → Segments → Data (upper layers)

Physical Layer — Media & Connecting Devices

GUIDED (WIRED) MEDIA

  • Twisted Pair (UTP/STP): Cheapest, most common. Cat5e/Cat6. Max ~100m. UTP unshielded, STP shielded.
  • Coaxial: Better shielding vs EMI. Used in cable TV/broadband. BNC/F-connectors.
  • Fiber Optic: Light pulses. Immune to EMI. Highest bandwidth. Single-mode = long distance (laser). Multi-mode = short distance (LED).

UNGUIDED (WIRELESS) MEDIA

  • Radio Waves: Omnidirectional, penetrates walls. WiFi, Bluetooth, AM/FM.
  • Microwaves: Directional, line-of-sight. Satellite, cellular, point-to-point links.
  • Infrared: Short range, line-of-sight. TV remotes. Cannot penetrate walls.
  • Satellite: Global coverage. High latency (~270ms GEO). LEO satellites lower latency.
  • Cellular: GSM (2G), CDMA, 3G, LTE (4G), 5G NR.

Ethernet & MAC Addresses
Ethernet Frame Format:
Preamble(7B) | SFD(1B) | Dst MAC(6B) | Src MAC(6B) | Type/Len(2B) | Data(46–1500B) | FCS(4B)
  • Preamble (7B): 10101010… alternating bits — clock sync
  • SFD (1B): 10101011 — Start Frame Delimiter
  • Dst/Src MAC (6B each): Hardware addresses
  • Type/Length (2B): ≥0x0600 = EtherType; less = length
  • Data (46–1500B): Payload. Min 46B needed for CSMA/CD collision window
  • FCS (4B): CRC-32 for error detection
  • Min total frame = 64 bytes (preamble/SFD excluded from size count)

MAC ADDRESS

  • 48-bit (6 octets) burned into NIC. Written in hex: AA:BB:CC:DD:EE:FF
  • First 24 bits = OUI (vendor). Last 24 bits = device-specific.
  • Unicast: LSB of first octet = 0
  • Multicast: LSB of first octet = 1
  • Broadcast: FF:FF:FF:FF:FF:FF — sent to all devices
Ethernet & OSI: Operates at L1 (Physical — signals/cables) and L2 (Data Link — MAC addressing + framing). LLC sublayer handles framing; MAC sublayer handles channel access.
Frame Processing: Switch reads Dst MAC → looks up MAC table → forwards out correct port (or floods if unknown).

Data Link Layer — Error Detection & Correction

ERROR TYPES

  • Single-bit error: One bit flipped (0↔1). Rare in serial links.
  • Burst error: Multiple consecutive bits corrupted. Common in real channels. Burst length = 1st to last corrupted bit.
  • Redundancy: Extra bits added for detection/correction. Trade-off: bandwidth overhead.

ERROR DETECTION METHODS

  • Parity (VRC): 1 bit appended so # of 1s = even (even parity). Detects odd-number bit errors only.
  • LRC (2D Parity): Parity for each row AND column. Can detect and correct 1-bit burst errors.
  • Checksum: Sum all data segments → 1's complement of sum. Receiver sums all + checksum = 0 (all 1s). Used in UDP, IP, TCP.
  • CRC: Most powerful. Treat message as polynomial, divide by generator. Remainder appended. Detects all burst errors < generator degree.

CRC — CYCLIC REDUNDANCY CHECK

  • Treat data as polynomial M(x), generator = G(x)
  • Append r zeros to M(x), divide by G(x) → remainder R(x)
  • Transmitted frame = M(x) – R(x) (XOR)
  • Receiver divides received frame by G(x) → remainder = 0 if no error
  • CRC-32 used in Ethernet; CRC-16 in HDLC; CRC-CCITT in X.25

HAMMING CODE (Error Correction)

  • Parity bits placed at positions 2⁰, 2¹, 2², 2³… (1,2,4,8,…)
  • For m data bits, need r parity bits where: 2^r ≥ m + r + 1
  • Hamming (7,4): 4 data bits + 3 parity bits → can correct 1-bit errors
  • Hamming Distance: # of bit positions two codewords differ
  • Min distance ≥ d+1 → detect d errors
  • Min distance ≥ 2d+1 → correct d errors

Multiple Access Protocols

RANDOM ACCESS

  • Pure ALOHA: Transmit anytime. Collision → wait random time, retransmit. Vulnerable time = 2Tₜ. Max efficiency ≈ 18.4% (1/2e)
  • Slotted ALOHA: Transmit only at slot start. Synchronization required. Vulnerable time = Tₜ. Max efficiency ≈ 36.8% (1/e)
  • CSMA: Sense channel before transmitting (Listen Before Talk).
    • 1-persistent: If busy, wait till idle then transmit immediately
    • Non-persistent: If busy, wait random time, sense again
    • p-persistent: If idle, transmit with prob p
  • CSMA/CD: Collision Detection — used in wired Ethernet (802.3). Abort on collision, send jam signal, binary exponential backoff. Min frame = 2×Tp×B.
  • CSMA/CA: Collision Avoidance — used in WiFi (802.11). IFS + random backoff + RTS/CTS optional. Avoids collision (can't detect in wireless).

CHANNELIZATION PROTOCOLS

  • FDMA — Each station gets dedicated frequency band. No collisions. Inefficient if bursty traffic.
  • TDMA — Each station gets a time slot in repeating cycle. Requires synchronization.
  • CDMA — All stations share freq+time, use unique chip codes (Walsh codes). Used in 3G. Orthogonal codes = no interference.

CONTROLLED ACCESS

  • Reservation: Stations reserve time slots in a mini-slot before data transmission.
  • Polling: Primary polls each secondary in turn. No collision but polling overhead.
  • Token Passing: Token circulates. Only token holder may transmit. Token Ring (802.5), Token Bus. Deterministic, no collision.
  • Piggybacking: ACK piggybacks on outgoing data frame in opposite direction — reduces overhead in bidirectional communication.

Noisy Channel — ARQ Protocols

KEY TERMS

  • Tₜ (Transmission time): Frame size (bits) ÷ Bandwidth (bps)
  • Tₚ (Propagation delay): Distance ÷ Speed of signal
  • a = Tₚ / Tₜ — key ratio for efficiency calculations

STOP AND WAIT ARQ

  • Send 1 frame → wait for ACK → send next. Window = 1.
  • Timer: if ACK not received in time, retransmit. Seq numbers: 0 and 1.
  • Simple but very inefficient on high-bandwidth/high-latency links.
η = 1 / (1 + 2a) [no errors] η = (1−P) / (1 + 2a) [frame error prob = P]

GO-BACK-N (GBN)

  • Sender can have W unacknowledged frames in pipeline.
  • Error → retransmit error frame + ALL subsequent frames (receiver discards out-of-order).
  • Max window: W ≤ 2ⁿ − 1 (n = seq number bits). Receiver window = 1.
If W < 1+2a: η = W / (1+2a) If W ≥ 1+2a: η = 1

SELECTIVE REPEAT (SR)

  • Retransmit ONLY the errored frame. Receiver buffers out-of-order frames.
  • More efficient than GBN. Both sender & receiver maintain windows.
  • Max window: W ≤ 2^(n−1)
If W < 1+2a: η = W / (1+2a) If W ≥ 1+2a: η = 1

Noiseless Channels — Elementary DLL Protocols
  • Simplex Protocol: One-direction only. Receiver always ready. No ACK. Theoretical.
  • Stop-and-Wait (noiseless): ACK from receiver controls flow. Sender waits before sending next.
  • Stop-and-Wait (noisy): Adds sequence numbers (0,1) and timer for lost frames/ACKs.
  • Piggybacking: Combine ACK with outgoing data → reduces overhead in bidirectional links.
  • Sliding Window: Generalization. GBN and SR are sliding window protocols.

Network Layer — IPv4 Addressing

IPv4 HEADER (min 20 bytes)

  • Version (4b): 4 for IPv4
  • IHL (4b): Header length in 32-bit words (min 5 = 20 bytes)
  • Total Length (16b): Header + Data, max 65535 bytes
  • Identification (16b): ID for fragmented datagrams
  • Flags (3b): DF (Don't Fragment), MF (More Fragments)
  • Fragment Offset (13b): Position of fragment × 8 (in bytes)
  • TTL (8b): Decremented at each router. Drop when = 0.
  • Protocol (8b): ICMP=1, TCP=6, UDP=17, OSPF=89
  • Header Checksum (16b): Error check for header only
  • Src / Dst IP (32b each): 4-byte addresses

Subnetting · ARP · RARP · ICMP · IGMP · IPv6

SUBNETTING (CIDR)

  • Borrow n bits from host → 2ⁿ subnets
  • Remaining h host bits → 2ʰ − 2 usable hosts (subtract net + broadcast)
  • Network addr: All host bits = 0 | Broadcast: All host bits = 1
  • Block size = 256 − subnet_mask_octet (addresses increment by block size)
Example /26: 64 IPs, 62 hosts, block size=64 Subnets: .0, .64, .128, .192

ARP & RARP

  • ARP: Maps IP → MAC. Broadcasts "who has IP X?" → target replies with its MAC. Result cached in ARP table (timeout ~20min).
  • RARP: Maps MAC → IP. Used by diskless workstations. Replaced by DHCP.
  • Proxy ARP: Router replies to ARP on behalf of remote host.
  • Gratuitous ARP: ARP for own IP — detects conflicts, updates neighbors' caches.

ICMP & IGMP

  • ICMP: Internet Control Message Protocol (IP Protocol 1). Reports errors: Destination Unreachable, TTL Exceeded, Redirect. Diagnostic: Echo (ping), Traceroute (TTL=1,2,3…)
  • IGMP: Internet Group Management Protocol (IP Protocol 2). Manages multicast group membership between hosts and local routers. Hosts send Join/Leave; routers query.

IPv6 KEY FACTS

  • 128-bit addresses, 8 groups of 4 hex: 2001:0db8::1
  • No checksum in header (offloaded to upper layers)
  • No fragmentation at routers (source does it via Path MTU Discovery)
  • No broadcast → uses multicast + anycast
  • Built-in IPSec. Auto-configuration (SLAAC). Simplified 8-field header (IPv4 has 13).
  • Loopback: ::1 | Link-local: fe80::/10 | Multicast: ff02::1
  • Transition: Dual Stack, Tunneling (6in4), NAT64/DNS64

Routing Algorithms & Protocols

ROUTING ALGORITHMS

  • Distance Vector (Bellman-Ford): Each router knows only its neighbors' distances. Shares routing table with neighbors periodically. Slow convergence. Count-to-infinity problem. Used by RIP.
  • Link State (Dijkstra): Each router builds complete topology map (LSA flooding). Runs Dijkstra's SPF algorithm. Fast convergence. Used by OSPF.
  • Static Routing: Manually configured. Zero routing overhead. Not adaptive to failures.
  • Dynamic Routing: Auto-learned via protocols. Adapts automatically to changes.
  • Shortest Path: Dijkstra — greedy algorithm. O((V+E) log V). Finds least-cost path.

TCP HEADER KEY FIELDS

  • Source Port (16b) + Destination Port (16b)
  • Sequence Number (32b) — byte stream position
  • Acknowledgment Number (32b) — next expected byte
  • Flags: URG, ACK, PSH, RST, SYN, FIN
  • Window Size (16b) — flow control (receive buffer size)
  • Checksum (16b) + Urgent Pointer (16b)

THREE-WAY HANDSHAKE

Connection Establishment:
1. Client → Server: SYN (seq=x)
2. Server → Client: SYN-ACK (seq=y, ack=x+1)
3. Client → Server: ACK (ack=y+1)
→ Connection established ✓
Four-way Termination:
→ FIN → ACK → FIN → ACK → closed

CONGESTION CONTROL

  • Slow Start: cwnd = 1 MSS, doubles each RTT until ssthresh
  • Congestion Avoidance: After ssthresh, increase cwnd by 1 MSS/RTT (linear)
  • Fast Retransmit: 3 duplicate ACKs → retransmit immediately (no timeout)
  • Fast Recovery: After 3 dup ACKs → halve ssthresh, skip slow start
  • QoS: Guarantees for bandwidth, latency, jitter, loss for prioritized traffic

Application Layer Protocols

DNS — Domain Name System

  • Translates domain names ↔ IP addresses. Port 53, UDP (queries) / TCP (zone transfers)
  • Hierarchy: Root (.) → TLD (.com, .in) → SLD (google) → Hostname (www)
  • Resolver → Root → TLD → Authoritative server (recursive resolution)
  • Records: A (IPv4), AAAA (IPv6), MX (mail), CNAME (alias), NS, PTR (reverse)

FTP — File Transfer Protocol

  • Port 21: Control connection (commands). Port 20: Data connection.
  • Active mode: Server initiates data conn from port 20 to client.
  • Passive mode: Client initiates both connections (firewall-friendly).
  • SFTP (SSH, port 22) and FTPS (FTP+TLS) are secure alternatives.

HTTP & WWW

  • HTTP port 80, HTTPS port 443 (TLS encrypted). Stateless protocol.
  • Methods: GET, POST, PUT, DELETE, PATCH, HEAD, OPTIONS
  • Status: 2xx OK · 3xx Redirect · 4xx Client error · 5xx Server error
  • HTTP/1.1: persistent connections · HTTP/2: multiplexing · HTTP/3: QUIC over UDP

EMAIL & REMOTE LOGIN

  • SMTP (port 25/587): Sends email client→server and server→server. Push protocol.
  • POP3 (port 110): Downloads & deletes from server. Not multi-device friendly.
  • IMAP (port 143): Keeps mail on server. Sync across devices. Preferred over POP3.
  • Telnet (port 23): Unencrypted remote login. Deprecated.
  • SSH (port 22): Encrypted remote login + file transfer (SFTP). Replaces Telnet.

Network Security

CRYPTOGRAPHY

  • Symmetric (Private Key): Same key for encrypt & decrypt. Fast. Key distribution problem. Examples: AES, DES, 3DES, RC4, Blowfish.
  • Asymmetric (Public Key): Public key encrypts; private key decrypts (or vice versa). Solves key distribution. Slower. Examples: RSA, ECC, Diffie-Hellman, ElGamal.
  • Hash Functions: One-way, fixed output. Integrity only. MD5 (128b, broken), SHA-1 (160b, deprecated), SHA-256 (256b, secure).
  • Digital Certificate / PKI: Certificate Authority (CA) signs public keys. Verify identity.

DIGITAL SIGNATURE

  • Sender: Hash(message) → encrypt with private key → signature
  • Receiver: Decrypt with sender's public key → compare hash
  • Provides: Authentication, Non-repudiation, Integrity
  • Does NOT provide confidentiality (encrypt separately for that)

IPSec

  • Network-layer security framework. Two sub-protocols:
  • AH (Authentication Header — IP protocol 51): Authentication + integrity. NO encryption.
  • ESP (Encapsulating Security Payload — IP protocol 50): Encryption + optional authentication.
  • Transport Mode: Protects payload only. IP header unchanged. Host-to-host.
  • Tunnel Mode: Protects entire packet. New outer IP header added. Used in VPNs.
  • IKE: Internet Key Exchange — negotiates keys & security associations. UDP 500.

FIREWALLS

  • Packet Filter: Stateless. Checks IP/port in header. Fast but limited.
  • Stateful Inspection: Tracks connection state (SYN/ACK). More secure.
  • Application-Level Gateway (Proxy): L7 full inspection. Most secure, slowest.
  • Circuit-Level Gateway: L5. Validates TCP handshake but doesn't inspect payload.
PGP (Pretty Good Privacy): Email encryption. Hybrid — symmetric key encrypts message; asymmetric (public key) encrypts the symmetric key. Provides: Confidentiality, Integrity, Authentication, Non-repudiation.
